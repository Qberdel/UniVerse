# Функционал проекта UniVerse V4

Краткое описание возможностей приложения и пользовательских сценариев.

## Назначение

Веб-клиент для системы учёта активности университетов и студентов (в т.ч. «АК» — активные коины), рейтингов ВУЗов, личного кабинета и витрины товаров за АК. Часть данных — демо на фронте, часть — через backend API.

## Общий интерфейс

- **Общий layout** (`RootLayout`): шапка с логотипом UniVerse, навигация, основная область, подвал.
- **Навигация**: Главная, Товары (страница каталога), Добавить активность, Профиль, Модерация. Корзина доступна со страницы товаров, отдельный маршрут `/cart` сохранён.

## Авторизация и доступ

### Регистрация (`/register`)

- Форма: имя, email, университет, пароль, подтверждение пароля.
- **Backend**: `POST /register` с телом JSON (`email`, `password`, при необходимости `name`, `university`).
- Базовый URL API задаётся в `url.txt` (первая строка, формат `API_BASE_URL=...`), подставляется при сборке Vite.
- При успехе: сохранение профиля в `localStorage`, переход на страницу входа.
- При ошибке: отображение сообщения от сервера (или запасного текста).

### Вход (`/login`)

- **Backend**: `POST /login` с `email` и `password`.
- При успехе: JWT сохраняется в **cookie**, флаг сессии в `localStorage`, переход в профиль.
- Ссылка на восстановление пароля и на регистрацию.

### Восстановление пароля (`/forgot-password`)

- Демо-сценарий: проверка email, имитация отправки ссылки без реального API.

### Доступ к профилю

- Маршрут `/profile` защищён: при отсутствии признака авторизации — редирект на `/register`.
- Учитывается наличие JWT в cookie и/или флаг регистрации в `localStorage`.

### Выход

- Очистка cookie с токеном, флага и профиля в `localStorage`, переход на `/login`.

## Профиль (`/profile`)

- Отображение данных пользователя из `localStorage` (имя, университет, email, аватар при наличии).
- **Backend**: запрос `GET /profile` с заголовком `Authorization: Bearer <JWT>` для проверки доступа (ответ пока может не содержать полного профиля с сервера).
- Редактирование: модальное окно — смена email и аватара (локально в `localStorage`).
- Два варианта отображения: «Кабинет» и «Компактный».
- **История заявок / активности** (демо на клиенте):
  - статусы: На проверке, Принято, Отклонено, Дополнить;
  - фильтры по вкладкам: Все, На проверке, Принято, Отклонено, Дополнить;
  - для статуса «Дополнить» — модальное окно с текстом дополнения; после отправки статус снова «На проверке».
- График истории АК (демо-данные).

## Главная страница (`/`)

- Сводные карточки метрик (демо).
- Вкладки: рейтинг ВУЗов и аналитика.
- Поиск и фильтр по категории по списку университетов (демо-данные).
- Карточки университетов с переходом к детальной странице.

## Университет (`/university/:id`)

- Одна страница-шаблон для любого `id` в URL; контент пока демо-статический (без подгрузки с API по идентификатору).

## Сравнение ВУЗов (`/compare`)

- Выбор двух университетов, демо-графики и подсказки (без реального API сравнения).

## Товары (`/menu`)

- Каталог товаров за АК: категории, поиск, добавление в локальную «корзину» (состояние на странице).
- Кнопка перехода в **Корзину** (`/cart`).

## Корзина (`/cart`)

- Список выбранных позиций (демо-данные при первой загрузке), удаление, итого, оплата АК (имитация), ссылка на товары при пустой корзине.

## Добавить активность (`/add-activity`)

- Форма/сценарий добавления активности (демо-интерфейс под будущий API).

## Модерация (`/moderator`)

- Интерфейс модератора (демо): заявки, действия, без привязки к реальному backend в описанной версии.

## Технические аспекты функционала

| Область | Реализация |
|--------|------------|
| API база | `url.txt` → константа при сборке → `fetch` в `src/app/lib/api.ts` |
| Регистрация / вход / профиль API | `register`, `login`, `profile` |
| JWT | Cookie (`universe_token` и связанные функции в `auth.ts`) |
| Профиль в браузере | `localStorage` (`profile.ts`) |
| Остальные экраны | преимущественно демо-данные и UI |

## Что не входит в текущую версию

- Полная синхронизация всех экранов с backend.
- Реальная оплата и склад товаров на сервере.
- Автогенерация страниц ВУЗов из API по `id` (шаблон готов, данные пока локальные).
